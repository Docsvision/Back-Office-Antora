.Чтобы {action} данные карточки:
. Установите флаг `*{act} карточки (XML структурированной части)`*
. Определите способ преобразования данных в поле _Преобразование_.
+
Преобразование применяется, чтобы при печати, экспорте или отправке по почте данные карточки были оформлены в удобном для восприятия виде.
+
Данный вид настраивается заранее в файле шаблона с расширением `.xslt`, использующем те или иные значения из исходного XML-кода карточки.
+
[NOTE]
====
XSL-шаблон должен быть заранее подготовлен в любом текстовом редакторе. Чтобы карточки могли использовать этот шаблон, специалист по внедрению системы должен подключить его в _Справочнике видов карточек_.
//Более подробные сведения о применении XSLT-шаблонов можно прочитать в отдельной https://{dv}.zendesk.com/entries/20913462-{dv}-1[статье].
====
+
Будет предложено сохранить данные карточки в файловой системе в виде XML-файла.

[start=6]
.Чтобы {action} основные файлы и подписи:
. В области _Файлы_ и _Подписи_ при помощи флагов выберите объекты, которые нужно {action}:
+
* Один или несколько основных файлов.
* Одну любую подпись. Можно {action} файл без выбора подписи.
+
. Определите формат {file} файла:
+
* *{act} основного файла и его электронной подписи отдельно* -- файл будет {done}, также отдельно будет {done} файл подписи в формате `.p7s`.
* *{act} основного файла и его электронной подписи в одном файле (.p7s)* -- файл и подпись будут объединены в один файл формата `.p7s`.
+
ifeval::["{action}" == "экспортировать"]
Простая подпись не может быть экспортирована.
+
endif::[]

+
. Чтобы {action} дополнительные файлы, приложенные к карточке, установите флаг `*{act} дополнительных файлов*`.
. Чтобы поместить результаты экспорта в ZIP-архив, установите флаг `*Архивировать выгруженные данные*`.
+
ifeval::["{action}" == "экспортировать"]
Данный параметр распространяется на все варианты экспорта, кроме экспорта в формат `.pdf`.
endif::[]
+
. Нажмите на кнопку image:buttons/export.png[Шестерёнка] *{btn}*.
+
ifeval::["{action}" == "отправить"]
В почтовой программе будет сформировано письмо, в тему которого будет автоматически подставлено название _основного_ файла.
+
В тело письма будет вставлен URL-адрес карточки с указанием полного пути к карточке.
+
Во вложение в зависимости от заданных параметров отправки будет добавлен файл формата `.p7s`, _основной_ файл и файл формата XML с данными карточки.
endif::[]
